# Guia da Live — 📺 Validação de Dados e Bibliotecas de Qualidade

## Objetivo da Live

Esta live transforma o tema `📺 Validação de Dados e Bibliotecas de Qualidade` em uma prática de engenharia de machine learning. O foco não é apenas apresentar conceitos, mas mostrar como decisões técnicas viram evidências: código executável, critérios de qualidade, documentação e relação com entrega.

## Pré-requisitos

- Ler o [README local](README.md) antes da sessão.
- Revisar o README da fase correspondente: [Fase 04](../../../README.md).
- Separar dúvidas sobre drift, observabilidade, validação de dados, governança, compliance, fairness e inferência causal.

## Agenda Sugerida

1. Abertura: conectar `📺 Validação de Dados e Bibliotecas de Qualidade` ao objetivo da Fase 04.
2. Leitura guiada do README local e identificação dos artefatos existentes.
3. Discussão técnica: decisões de engenharia, riscos e critérios de qualidade.
4. Atividade prática ou desenho de solução em pequenos grupos.
5. Fechamento com evidências que devem aparecer no Tech Challenge.

## Demonstração ou Discussão Principal

Execute e discuta o `exercicio.py` local, destacando entradas, saídas, métricas e decisões de implementação.

Durante a condução, explicite quais decisões são reversíveis, quais são arquiteturais e quais precisam aparecer como justificativa técnica no projeto.

## Perguntas Orientadoras

- Qual problema de engenharia este material resolve dentro do ciclo de vida de ML?
- Que evidência mostra que a solução é reproduzível e não apenas um experimento local?
- Quais métricas, logs ou documentos sustentam a decisão técnica?
- Que risco aparece se este tema for ignorado no Tech Challenge?

## Checkpoints de Aprendizagem

- O estudante consegue explicar o papel do tema na Fase 04.
- O estudante identifica entradas, saídas, dependências e limitações do material.
- O estudante conecta a discussão a uma decisão concreta de produto, dados, modelo ou operação.

## Conexão com o Tech Challenge

Use esta live para fortalecer a entrega que precisa criar evidências de que o sistema continua correto, seguro, auditável e justificável ao longo do tempo. Ao final, cada grupo deve registrar uma decisão técnica, uma evidência esperada e uma pendência realista para o projeto.
