# Atividade do Grupo — 📖 Grupo de Estudos 1 — Clean Code e Estrutura

## Contexto do Problema

O grupo precisa transformar `📖 Grupo de Estudos 1 — Clean Code e Estrutura` em uma ação concreta do projeto. A atividade é colaborativa: todos devem sair com uma decisão registrada e uma evidência que possa ser revisada.

## Tarefa

Preencha uma matriz simples com quatro colunas: decisão, justificativa, evidência e risco. Use o tema do encontro como filtro para evitar decisões genéricas.

## Passos da Atividade

1. Cada integrante propõe uma melhoria relacionada ao tema.
2. O grupo escolhe uma melhoria pela combinação de valor, esforço e risco.
3. O grupo define como vai provar que a melhoria funcionou.
4. O grupo registra o que não será feito nesta rodada.
5. O grupo agenda a evidência para a próxima revisão.

## Evidências Esperadas

- Matriz de decisão preenchida.
- Link para arquivo, notebook, script, diagrama ou README afetado.
- Critério de aceite verificável.
- Pendência realista para continuidade.

## Perguntas para Reflexão

- A decisão melhora o projeto ou apenas adiciona complexidade?
- Outra equipe conseguiria entender a justificativa?
- O risco tratado é técnico, operacional, regulatório ou de produto?

## Como Relacionar ao Tech Challenge

A atividade deve fortalecer a capacidade do grupo de permitir que outra pessoa reconstrua o ambiente, os dados, o modelo e as evidências sem depender de conhecimento tácito. Registre apenas orientação e evidência pública; não publique dados privados nem respostas fechadas.

## Critérios de Autoavaliação

- A decisão é pequena o suficiente para ser executada.
- O critério de aceite é observável.
- O grupo consegue explicar o trade-off.
- A documentação resultante ajuda a apresentação final.
